## Slide 02: What It Is
type: content

### heading
One Engine, Every Modality

### body
- Text, vision, and audio share a single low-latency runtime
- Routes each request to the smallest model that can answer it
- Streams first tokens in under fifty milliseconds
- Scales from a laptop to a fleet without code changes

## Slide 04: How It Works
type: process

### heading
Request to Response

### body
- Receive: accept text, image, or audio
- Route: pick the cheapest capable model
- Infer: stream tokens as they generate
- Trace: log every hop for replay
