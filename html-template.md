# HTML Presentation Template

Reference architecture for generating slide presentations. Every presentation follows this structure.

## Base HTML Structure

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Presentation Title</title>

    <!-- Fonts: use Fontshare or Google Fonts — never system fonts -->
    <link rel="stylesheet" href="https://api.fontshare.com/v2/css?f[]=...">

    <!-- Icons: Lucide (CDN, no download needed) -->
    <script src="https://unpkg.com/lucide@latest/dist/umd/lucide.min.js"></script>

    <style>
        /* ===========================================
           CSS CUSTOM PROPERTIES (THEME)
           Change these to change the whole look
           =========================================== */

            --text-secondary: #9ca3af;

        <h1 class="reveal">Presentation Title</h1>
        <p class="reveal">Subtitle or author</p>
    </section>

    <section class="slide">
        <div class="slide-content">
            <h2 class="reveal">Slide Title</h2>

            }

```

## Icons: Lucide (Default Library)

**Never use emoji as icons** — they look unprofessional ("AI slop"). Use Lucide icons via CDN instead.

Setup (already in the base HTML template above):
```html
<script src="https://unpkg.com/lucide@latest/dist/umd/lucide.min.js"></script>
```

Usage:
```html
<div class="feature-icon"><i data-lucide="server"></i></div>
```

Initialize after DOM is ready:
```javascript
if (typeof lucide !== 'undefined') lucide.createIcons();
```

CSS for icon styling (adapt colors to theme):
```css
.feature-icon {
    color: var(--accent);
    display: flex; align-items: center;
    margin-bottom: clamp(0.3rem, 0.8vw, 0.6rem);
}
.feature-icon svg {
    width: clamp(28px, 3.5vw, 40px);
    height: clamp(28px, 3.5vw, 40px);
    stroke: var(--accent);
    stroke-width: 1.5;
}
```

**Sizing:** Lucide icons are line-drawn SVGs, not emoji — size them generously (28–40px). They serve as visual anchors for cards, not inline text decorations. `stroke-width: 1.5` keeps lines refined at larger sizes.

**Theme adaptation:** Style icons to match the chosen aesthetic, not one-size-fits-all:

| Theme type | Icon color | Extra effects |
|-----------|-----------|--------------|
| Dark / Neon / Cyber | Accent color (cyan, magenta, etc.) | `filter: drop-shadow(0 0 6px ...)` for glow |
| Light / Clean / Modern | Muted accent or dark gray | None — clean lines are enough |
| Pastel / Playful | Accent tint, slightly thicker stroke | Optional soft `opacity: 0.8` |
| Editorial / Vintage | Dark ink color or sepia tone | None — let typography dominate |

## Required JavaScript Features

Every presentation must include:

1. **SlidePresentation Class** — Main controller with:
   - Keyboard navigation (arrows, space, page up/down)
   - Touch/swipe support
   - Mouse wheel navigation
   - Progress bar updates
   - Navigation dots

2. **Intersection Observer** — For scroll-triggered animations:
   - Add `.visible` class when slides enter viewport
   - Trigger CSS transitions efficiently

3. **Optional Enhancements** (match to chosen style):
   - Custom cursor with trail
   - Particle system background (canvas)
   - Parallax effects
   - 3D tilt on hover
   - Magnetic buttons
   - Counter animations
   - **Image lightbox** (auto-included when presentation has images — see "Image Lightbox" section below)

4. **Inline Editing** (only if user opted in during Step 2 — skip entirely if they said No):
   - Edit toggle button (hidden by default, revealed via hover hotzone or `E` key)
   - Press `E` to enter edit mode, `Esc` to save & exit
   - Auto-save to localStorage on exit
   - See "Inline Editing Implementation" section below

## Inline Editing Implementation (Opt-In Only)

**If the user chose "No" for inline editing in Phase 1, do NOT generate any edit-related HTML, CSS, or JS.**

**Do NOT use CSS `~` sibling selector for hover-based show/hide.** The CSS-only approach (`edit-hotzone:hover ~ .edit-toggle`) fails because `pointer-events: none` on the toggle button breaks the hover chain: user hovers hotzone -> button becomes visible -> mouse moves toward button -> leaves hotzone -> button disappears before click.

**Required approach: JS-based hover with 400ms delay timeout.**

HTML:
```html
<div class="edit-hotzone"></div>
<button class="edit-toggle" id="editToggle" title="Edit mode (E)">✏️</button>
```

CSS (visibility controlled by JS classes only):
```css
/* Do NOT use CSS ~ sibling selector for this!
   pointer-events: none breaks the hover chain.
   Must use JS with delay timeout. */
.edit-hotzone {
    position: fixed; top: 0; left: 0;
    width: 80px; height: 80px;
    z-index: 10000;
    cursor: pointer;
}
.edit-toggle {
    opacity: 0;
    pointer-events: none;
    transition: opacity 0.3s ease;
    z-index: 10001;
}
.edit-toggle.show,
.edit-toggle.active {
    opacity: 1;
    pointer-events: auto;
}
```

JS (three interaction methods):
```javascript
// 1. Click handler on the toggle button
document.getElementById('editToggle').addEventListener('click', () => {
    editor.toggleEditMode();
});

// 2. Hotzone hover with 400ms grace period
const hotzone = document.querySelector('.edit-hotzone');
const editToggle = document.getElementById('editToggle');
let hideTimeout = null;

hotzone.addEventListener('mouseenter', () => {
    clearTimeout(hideTimeout);
    editToggle.classList.add('show');
});
hotzone.addEventListener('mouseleave', () => {
    hideTimeout = setTimeout(() => {
        if (!editor.isActive) editToggle.classList.remove('show');
    }, 400);
});
editToggle.addEventListener('mouseenter', () => {
    clearTimeout(hideTimeout);
});
editToggle.addEventListener('mouseleave', () => {
    hideTimeout = setTimeout(() => {
        if (!editor.isActive) editToggle.classList.remove('show');
    }, 400);
});

// 3. Hotzone direct click
hotzone.addEventListener('click', () => {
    editor.toggleEditMode();
});

// 4. Keyboard shortcuts: E to enter edit mode, Esc to save & exit
document.addEventListener('keydown', (e) => {
    if ((e.key === 'e' || e.key === 'E') && !e.target.getAttribute('contenteditable')) {
        editor.toggleEditMode();
    }
    if (e.key === 'Escape' && editor.isActive) {
        editor.toggleEditMode(); // auto-saves on exit
    }
});
```

## Image Pipeline (Skip If No Images)

If user chose "No images" in Phase 1, skip this entirely. If images were provided, process them before generating HTML.

**Dependency:** `pip install Pillow`

### Image Processing

```python
from PIL import Image, ImageDraw

# Circular crop (for logos on modern/clean styles)
def crop_circle(input_path, output_path):
    img = Image.open(input_path).convert('RGBA')
    w, h = img.size
    size = min(w, h)
    left, top = (w - size) // 2, (h - size) // 2
    img = img.crop((left, top, left + size, top + size))
    mask = Image.new('L', (size, size), 0)
    ImageDraw.Draw(mask).ellipse([0, 0, size, size], fill=255)
    img.putalpha(mask)
    img.save(output_path, 'PNG')

# Resize (for oversized images that inflate HTML)
def resize_max(input_path, output_path, max_dim=1200):
    img = Image.open(input_path)
    img.thumbnail((max_dim, max_dim), Image.LANCZOS)
    img.save(output_path, quality=85)
```

| Situation | Operation |
|-----------|-----------|
| Square logo on rounded aesthetic | `crop_circle()` |
| Image > 1MB | `resize_max(max_dim=1200)` |
| Wrong aspect ratio | Manual crop with `img.crop()` |

Save processed images with `_processed` suffix. Never overwrite originals.

### Image Placement

**Copy all used images into `assets/` inside the presentation folder, then reference with relative paths.** This ensures the presentation is portable — zip the folder and it works anywhere.

```bash
# During generation, copy images to the output folder
mkdir -p my-presentation/assets
cp /path/to/user/screenshot.png my-presentation/assets/screenshot.png
cp /path/to/user/logo.png my-presentation/assets/logo.png
```

```html
<!-- Always use relative paths from the HTML file -->
<img src="assets/logo_round.png" alt="Logo" class="slide-image logo">
<img src="assets/screenshot.png" alt="Screenshot" class="slide-image screenshot">
```

**Never reference absolute paths or paths outside the presentation folder** — the recipient won't have those files.

```css
.slide-image {
    max-width: 100%;
    max-height: min(50vh, 400px);
    object-fit: contain;
    border-radius: 8px;
}
.slide-image.screenshot {
    border: 1px solid rgba(255, 255, 255, 0.1);
    border-radius: 12px;
    box-shadow: 0 8px 32px rgba(0, 0, 0, 0.3);
}
.slide-image.logo {
    max-height: min(30vh, 200px);
}
```

**Adapt border/shadow colors to match the chosen style's accent.** Never repeat the same image on multiple slides (except logos on title + closing).

**Placement patterns:** Logo centered on title slide. Screenshots in two-column layouts with text. Full-bleed images as slide backgrounds with text overlay (use sparingly).

### Image Lightbox

**When a presentation includes images, always include the lightbox.** Users need to zoom in on screenshots with small text. Skip only for logo-only presentations.


JS:
```javascript
class Lightbox {
    constructor() {
        this.overlay = document.getElementById('lightbox');
        this.img = this.overlay.querySelector('img');
        this.isOpen = false;

        // Attach click-to-zoom on all slide images
        document.querySelectorAll('.slide-image').forEach(img => {
            img.addEventListener('click', (e) => {
                e.stopPropagation();
                this.open(img.src, img.alt);
            });
        });

        this.overlay.addEventListener('click', () => this.close());
        this.overlay.querySelector('.lightbox-close').addEventListener('click', (e) => {
            e.stopPropagation();
            this.close();
        });

        document.addEventListener('keydown', (e) => {
            if (e.key === 'Escape' && this.isOpen) this.close();
        });
    }

    open(src, alt) {
        this.img.src = src;
        this.img.alt = alt;
        this.overlay.classList.add('active');
        this.isOpen = true;
    }

    close() {
        this.overlay.classList.remove('active');
        this.isOpen = false;
    }
}

// Initialize after SlidePresentation
new Lightbox();
```

**Adapt `.lightbox-close:hover` color** to match the chosen style's accent variable.

## Page Number (Always On)

A subtle page indicator pinned to the bottom-right of every slide, showing `current / total`. Updated automatically by JS when slides change.

HTML (place after nav dots, before slides):
```html
<div class="page-number" id="pageNumber">1</div>
```

CSS:
```css
.page-number {
    position: fixed;
    bottom: clamp(8px, 1.2vh, 16px);
    right: clamp(12px, 2vw, 24px);
    z-index: 999;
    font-family: var(--font-body);
    font-size: clamp(0.6rem, 0.9vw, 0.75rem);
    letter-spacing: 0.05em;
    /* Dark themes → white text; Light themes → dark text */
    color: rgba(255, 255, 255, 0.35);  /* dark theme default */
    /* color: rgba(0, 0, 0, 0.3);      /* light theme alternative */
    font-variant-numeric: tabular-nums;
}
@media (max-height: 600px) {
    .page-number { display: none; }
}
```

JS (add inside SlidePresentation or the updateUI function):
```javascript
const pageNum = document.getElementById('pageNumber');
function updatePageNumber(current, total) {
    if (pageNum) pageNum.textContent = current + 1;
}
// Call updatePageNumber(current, slides.length) whenever slide changes
```

**Styling rules:**
- Same opacity/color logic as `.slide-footer` — blend in, don't stand out
- `font-variant-numeric: tabular-nums` keeps numbers from jumping as digits change
- Hidden at `max-height: 600px` (same as footer)

## File Structure

**No media** — single file is enough:
```
presentation.html
```

**With images/videos** — dedicated folder, everything inside:
```
my-presentation/
├── index.html       # The presentation (all CSS/JS inline)
└── assets/          # All images, videos, GIFs
    ├── screenshot.png
    ├── logo.png
    └── demo.mp4
```

**Sharing:** Zip the entire folder. Recipient unzips and opens `index.html`.
