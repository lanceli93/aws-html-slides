# aws-html-slides — Feature Overview

> Draft summary of what the skill currently does, for later editing.
> Generated from the skill files + QA demos. Verify details against the
> source before treating any line as authoritative.

The skill generates **zero-dependency, animation-rich HTML presentations** that
run entirely in the browser — a single self-contained `index.html` (all CSS/JS
inline) plus an `assets/` folder for media. Double-click to open; no build step,
no server. Share by zipping the folder.

---

## 1. Two workflows

| Path | Flow |
|------|------|
| **New Presentation** | Setup → Topic → Style → fill `content.md` → review → generate HTML |
| **PPT Conversion** | Setup → `.pptx` path → Style + extract + generate (one step) |

- **Setup** asks (one prompt, bilingual): Language (EN/中文), Mode (new / convert),
  Inline editing (yes/no), Page count (Short 5–10 / Medium 10–20 / Long 20+).
- PPT conversion uses `scripts/extract-pptx.py` (via `uv run --with python-pptx`)
  to pull text, images, and speaker notes out of a `.pptx`.
- `content.md` is generated by `scripts/gen-content.py` (a template, NOT AI) and
  filled in by the user; the skill reviews it before generating.

## 2. Core principles (non-negotiable)

- **Zero dependencies** — single HTML file, inline CSS/JS, no npm/build.
- **Viewport fitting** — every `.slide` is exactly `100vh`/`100dvh`, `overflow:hidden`;
  content never scrolls within a slide. All sizing uses `clamp()`. Height
  breakpoints at 700/600/500px. Content that exceeds limits is split across slides.
- **Distinctive design** — avoid generic "AI slop"; commit to a cohesive aesthetic.
- **No emoji as icons** — use Lucide icons (inlined SVG when offline).
- **AWS disclaimer** on every slide (fixed footer) + a `current/total` page number.

## 3. Style presets

13 curated presets (defined in `STYLE_PRESETS.md`), spanning dark / light /
specialty themes — e.g. Bold Signal, Electric Studio, Neon Cyber, Terminal Green,
Swiss Modern, **re:Invent Keynote** (the most developed, with a dedicated
`layout-reinvent.md` of 10 layout patterns: title, pill cards, stats grid, split
divider, pipeline, quote, big-number, comparison, process flow, timeline zigzag).
Previews are pre-built HTML files the user can open and pick from.

## 4. Rich content blocks (usable in any slide)

| Block | What it does | Tech | Reference |
|-------|--------------|------|-----------|
| `### chart` | Data charts: line, bar, doughnut, pie, radar, polarArea | Chart.js (CDN) | `chart-reference.md` |
| `### table` | Comparison tables from markdown table syntax | themed CSS | `html-template.md` |
| `### diagram` | Flowchart, AWS architecture, sequence/UML/ER, network, mindmap | draw.io CLI → PNG in `assets/`, embedded as image + lightbox | `diagram-reference.md` |
| media | Images / video, click-to-enlarge lightbox | inline `<img>`/`<video>` | `html-template.md` |

- **Diagrams** require the draw.io desktop CLI (`brew install --cask drawio`) and
  the drawio skill; if missing, the skill falls back to a CSS layout. Export with
  `-x -f png -s 2 -t` (2× for sharp lightbox zoom, transparent bg). Theme-match
  colors; on dark themes set edge-label backgrounds or they render as white boxes.

## 5. Built-in JS features (every deck)

- **Slide engine** — keyboard (arrows/space/PageUp-Down/Home/End), wheel, touch
  swipe, nav dots, progress bar, IntersectionObserver-driven reveal animations.
- **Overview mode** (always on) — `Esc` opens a PPT-style thumbnail grid; click /
  arrows+Enter to jump. Thumbnails are live-DOM clones scaled down; Chart.js
  canvases are rasterized to images so they aren't blank. (Known fix: force the
  revealed state on clones so not-yet-viewed slides aren't blank.)
- **Inline editing** (opt-in) — `E` to edit, click any text, `Esc` to save & exit;
  autosaves to `localStorage`.
- **Save-to-file** — `Ctrl/Cmd+S` writes edits back into `index.html` via the File
  System Access API (Chrome/Edge), using a DOMParser diff-and-patch so charts /
  scripts / structure are preserved. Survives cache clears and sharing.
- **Image lightbox** — auto-included when a deck has images/diagrams.

## 6. 3D & motion effects (`effects-reference.md` + `demos/`)

A license-checked menu of ways to add 3D/motion while staying offline. **Three
distinct 3D techniques**, each with a working demo:

1. **Three.js real-3D** (`demos/threejs-hero/`) — genuine WebGL: emissive morphing
   core, fresnel rim, orbiting lights, particle cloud, hand-rolled inline bloom
   (no ESM addons), ACES tone mapping. Vendor `three.min.js` **r160** (last UMD/
   global build) locally for offline use.
2. **CSS 3D** (`demos/css-3d-hero/`) — pure-CSS rotating cube + 2.5D parallax via
   `transform-style: preserve-3d` + `perspective`. Watch the *flattening gotcha*:
   no opacity/filter/overflow/clip/mask/blend on the preserve-3d element — put
   those on children.
3. **Pointer-driven 3D tilt** (`demos/pointer-tilt/`) — cards tilt toward the
   cursor (capped + eased in one rAF loop), inner layers at different `translateZ`
   for parallax, specular sheen tracking the pointer.

Other inline effects: hand-written **WebGL shader background** (`webgl-nebula/`),
hand-written **2D-canvas constellation** (`particles-network/`), **GSAP** kinetic
type + content micro-effects (`gsap-title/`, `gsap-micro/`; GSAP 3.13 free for
commercial, vendored), `@property` spinning conic gradients, View Transitions.

> Status: these effects are a **deliberate, per-deck menu** — NOT yet wired into
> the default generation flow. `demos/qa-runs/nova-core/` shows all three 3D
> techniques integrated into one re:Invent deck (with per-slide animated
> backgrounds and an IntersectionObserver that runs each effect only on its slide).

Licensing red-lines are documented (Spline export = paid, Shadertoy = CC BY-NC-SA,
LYGIA = paid, etc.); always-safe: Three.js, GSAP, tsParticles, lottie-web, Lucide,
MIT `snoise`, CC0 asset hubs.

## 7. Output structure

```
my-presentation/
├── index.html        # the deck — all CSS/JS inline, zero deps
├── assets/           # images, video, diagram PNGs (relative paths only)
└── vendor/           # only if a vendored lib is used (three.min.js, gsap.min.js)
```

Generation uses **parallel sub-agents for 8+ slides** (a framework agent + one
agent per ~5-slide batch, concatenated), single-agent for 1–7.

## 9. Demos & QA decks (this repo, `demos/`)

- **Effect demos** — one per technique (above), each offline + self-contained.
- **`demos/qa-runs/`** — full decks built end-to-end to QA skill features:

  - `serverless-arch/` — real drawio diagram pipeline (AWS arch + flowchart)
  - `launch-deck/` — inline editing + save-to-file + 6 layout types
